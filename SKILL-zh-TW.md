# 這篇能發嗎？

> 貼上草稿，拿到 0-100 分 + 逐句診斷 + 改寫版。

## 觸發條件

當使用者說以下任一詞時啟動：
- 「評分」「這篇能發嗎」「幫我看看這篇」「可以發嗎」
- 「打個分」「檢查一下」「這樣寫可以嗎」
- 或直接貼出一段文字並期望得到發文建議

## 首次使用：個人化校準

如果 skill 資料夾中沒有 `MY_PROFILE.md`，先跑校準再評分。逐題詢問：

### 問題 1（必填）
「你主要在哪個平台發文？」
- Threads / Instagram / X (Twitter) / LinkedIn / 其他

→ 決定格式規則和殺觸及偵測。每個平台的演算法不同，懲罰的東西也不同。

### 問題 2（必填）
「你的內容主題是什麼？列 3-5 個關鍵字（例如：AI、健身、育兒、投資）」

→ 取代預設主題清單，成為你的品牌一致性基準。不在你主題範圍內的貼文，品牌一致性分數會降低。

### 問題 3（必填）
「你發文的主要目標是什麼？」
- 漲粉
- 賣產品或服務
- 建立專業形象
- 純分享 / 記錄自己的過程

→ 調整評分權重：
- 漲粉 → Hook 力和對話引力權重提高
- 賣東西 → 行動觸發權重提高
- 建立專業 → 原創密度權重提高
- 純分享 → 情緒共鳴權重提高

### 問題 4（選填）
「你的受眾是誰？用一句話描述。」
（例如：「30 多歲學寫程式的爸媽」或「台灣的小型創業者」）

→ 調整語氣和情緒共鳴的判定標準。

### 問題 5（選填）
「貼一篇你寫過最滿意的貼文——你最驕傲的那篇。」

→ 分析它來反推你的寫作風格：句子長度、語氣、結構模式。這比任何問卷都準。

### 校準完成後

將結果存為 skill 資料夾中的 `MY_PROFILE.md`：

```markdown
# 我的評分設定

- 平台：[回答]
- 主題：[回答]
- 目標：[回答]
- 受眾：[回答或「未指定」]
- 風格樣本：[回答或「未提供」]
- 校準日期：[日期]

## 衍生設定
- 品牌主題：[Q2 的清單]
- 權重調整：[從 Q3 衍生]
- 語氣基準：[從 Q5 衍生或預設]
- 平台特定規則：[從 Q1 衍生]
```

告訴使用者：「設定已儲存。隨時說『重新校準』就能修改。現在貼上你的第一篇草稿吧。」

後續使用時，靜默讀取 `MY_PROFILE.md` 後直接評分。

## 評分流程

收到草稿後，依序執行三步：

### Step 1：評分（0-100）

依 8 個維度打分，每個維度獨立評分後加總：

| 維度 | 分數 | 評分標準 |
|------|------|---------|
| **Hook 力** | 0-15 | 前兩句能不能讓人停下拇指？爭議主張（15）> 反直覺（12）> 系列感「又XX了」（12）> 問句（10）> 數字開頭（8）> 沒有 Hook（0-3） |
| **對話引力** | 0-20 | 讀完後想不想回覆？開放問句收尾（18-20）> 爭議立場引發正反方（15-18）> 自我評估框架「你是哪一級」（12-15）> 資訊給完沒有對話空間（0-5） |
| **情緒共鳴** | 0-15 | 有沒有觸動真實感受？痛點堆疊＋場景還原（15）> 真實踩坑經驗（13-14）> 共鳴型洞察（10-12）> 有趣但淺（6-9）> 冷冰冰的資訊（0-5） |
| **原創密度** | 0-10 | 有沒有作者獨有的觀點/經驗/數據？一手實戰（8-10）> 個人觀點＋案例（5-7）> 整理公開資訊（2-4）> 純轉述（0-1） |
| **格式節奏** | 0-10 | 換行、斷句、閱讀節奏。短句交錯＋適當換行（8-10）> 段落分明（5-7）> 文字牆（0-3） |
| **行動觸發** | 0-10 | 讀者有沒有理由儲存/分享？有可執行步驟值得存（8-10）> 有金句值得轉（5-7）> 看完就走（0-3） |
| **品牌一致性** | 0-10 | 是否在使用者定義的主題內＋符合語氣？完全吻合（8-10）> 相關但偏（4-7）> 不在主題內（0） |
| **殺觸及偵測** | 10 減扣分 | 從 10 分開始，每發現一個殺觸及因子扣 2-3 分，最低 0 分 |

**殺觸及因子清單**（每命中一項扣 2-3 分）：
- 互動釣魚：「留言 YES」「回覆 1 拿連結」「同意嗎？」
- 商業推銷感：「限時」「優惠」「點連結」「推薦碼」
- 回收內容感：看起來像從 IG/FB 貼過來的
- 文字牆：超過 5 行沒換行
- 公關/企業語氣
- 只有資訊沒有觀點（像維基百科）
- 外部連結（Threads 演算法壓外連貼文）

**各平台特定殺觸及因子：**
- Threads：外部連結、hashtag 濫用（>5 個）、@mention 濫用
- Instagram：caption 內放連結（反正也不能點）、互動釣魚
- X/Twitter：蹭串感、太多 hashtag
- LinkedIn：emoji 過量、「I'm humbled」類型廢話

**進階 Hook 模式：**

系列感型：用「又XX了」「今天XX又瘋了」建立可重複的欄目 Hook。讀者看到就知道「又有好東西了」，養成追蹤習慣。

痛點堆疊法：不只講一個痛點，一口氣列 3-7 個具體場景，讓讀者覺得「這個人完全懂我」。堆疊之後接場景還原，情緒共鳴拉滿。

**分數區間解讀：**
- 80-100：直接發，高機率表現好
- 70-79：小改即可，調整 Hook 或收尾
- 50-69：結構有問題，需要改寫
- 0-49：建議換個角度重新來

### Step 2：逐句診斷

對草稿的每一句標記：

- 🟢 **強句**：有 Hook 力 / 引發回覆 / 情緒共鳴 / 值得截圖
- 🟡 **可改**：方向對但表達弱 / 可以更口語 / 節奏可調
- 🔴 **殺觸及**：會降權 / 太正式 / 互動釣魚 / 無對話空間

每個標記附一行原因說明。

範例：
```
🟢「兩個月前我把 Claude 徹底融入工作流後，發現一件事。」
   → 強 Hook：第一人稱＋時間框架＋懸念（發現什麼？）

🟡「如果你還在手動寫腳本、手動搬數據，那你在 AI 時代依然是在做體力活。」
   → 觀點好但太長，建議拆成兩句增加節奏感

🔴「歡迎追蹤我獲取更多 AI 工具分享！」
   → 殺觸及：公關語氣＋無對話空間。改成問句：「你目前最痛的手動流程是什麼？」
```

### Step 3：改寫版

基於診斷結果，產出一個目標 70 分以上的改寫版。改寫規則：

1. **Hook 必改**：如果原文 Hook < 10 分，重寫前兩句
2. **收尾必改**：如果沒有對話觸發，加上開放問句或自我評估框架
3. **格式重排**：短句優先、適當換行、關鍵句獨立成行
4. **口語化**：把書面語改成朋友聊天的語氣
5. **保留原意**：改的是表達方式，不是觀點內容

改寫版同時標注：
- 建議發布格式：純文字 / 圖文 / self-reply thread
- 如果適合拆 thread：給出拆法建議（幾則、每則重點）

## Self-Reply Thread 建議

當以下任一條件成立，建議使用 self-reply thread：
- 內容超過 300 字（超過單篇最佳閱讀長度）
- 有 3 個以上步驟/要點
- 可以用「Part 1/N」結構
- 主題適合層遞展開（觀點 → 案例 → 做法 → 互動問句）

三種 thread 模板：

**模板 A 層遞式**（觀點型）：
```
主帖：爭議性 Hook + 核心主張
 ↳ 回覆1：為什麼這樣說（理由/數據）
   ↳ 回覆2：具體怎麼做（步驟）
     ↳ 回覆3：互動問句（邀請討論）
```

**模板 B 連載式**（教學型）：
```
主帖：Hook + Part 1/5
 ↳ 自回：Part 2/5
   ↳ 自回：Part 3/5
     ↳ 自回：Part 4/5
       ↳ 自回：Part 5/5 + 互動問句
```

**模板 C 懸念式**（引流型）：
```
主帖：強 Hook +「整理在留言 👇」
 ↳ 自回1-N：實際內容
 ↳ 最後一則：互動問句 + 追蹤邀請
```

## 演算法參考（評分依據）

評分參考以下已知的演算法因子：

評分依據分兩類證據：**Threads/Meta 官方** 和 **X (Twitter) 開源碼**。只在單一平台確認的因子會標明，兩平台共通或有跨平台研究支持的標「通用」。

### Threads / Meta（官方）

| 因子 | 權重 | 說明 | 來源 |
|------|------|------|------|
| 互動速度 | 最高 | 發布後 30-60 分鐘內的互動量決定推播 | 業界共識，多項 Threads 研究 |
| 回覆深度 | 極高 | 回覆（特別是回覆的回覆）是最強的互動信號 | [Meta Transparency Center](https://transparency.meta.com/features/approach-to-ranking/content-distribution-guidelines/engagement-bait) |
| 儲存＋分享 | 高 | 代表高價值，比按讚重要 | 業界共識 |
| 互動釣魚懲罰 | 降權 | 「留言 YES」「回覆 1 拿連結」等會被降權 | [Meta Transparency Center](https://transparency.meta.com/features/approach-to-ranking/content-distribution-guidelines/engagement-bait)（官方）|
| 原創性 | 中高 | 回收內容被降權，原創內容被加分 | 業界共識、Meta 聲明 |
| 內容新鮮度 | 中 | 新貼文有推播加成 | 業界共識 |

### X / Twitter（開源碼 + 馬斯克發言）

以下因子已在 X 上透過開源碼或馬斯克公開發言**確認**。在 Threads 上原則相似（兩者都是動態牆社群平台），但 **Meta 未官方確認**。

| 因子 | 權重 | 說明 | 來源 |
|------|------|------|------|
| 停留時間 | 最高 | 演算法目標是最大化「不後悔的使用者停留秒數」。長內容（停留 2 分鐘以上）權重 +10 | [馬斯克推文](https://x.com/elonmusk/status/1875355425601999255)、[X 開源碼](https://github.com/twitter/the-algorithm) |
| 原生內容 | 高 | 平台原生的文字和影片，表現優於外部連結 | [X 開源碼](https://github.com/twitter/the-algorithm) |
| 長內容 | 高 | 長文和長影片觸及更好，因為自然增加停留時間 | [X 開源碼](https://github.com/twitter/the-algorithm) |
| 影片 > 圖片 | 中高 | 影片每秒承載的資訊量比靜態圖片多 | [X 開源碼](https://github.com/twitter/the-algorithm)、Buffer 4500 萬篇分析 |
| 不要用 Hashtag | — | 「請不要再用 hashtag 了，系統已經不需要它了，而且很醜。」 | [馬斯克推文](https://x.com/elonmusk/status/1869070358210572306) |
| 正向/建設性 | 中 | 負面內容被壓。演算法追求「不後悔的」使用者停留 | [馬斯克推文](https://x.com/elonmusk/status/1875355425601999255) |

## 發布時段建議

| 時段 | 建議 |
|------|------|
| 平日 7-10am | 最佳（全球數據＋通勤時段） |
| 平日 12-1pm | 不錯（午休滑手機） |
| 平日 9-10pm | 可試（晚間放鬆） |
| 週六全天 | 避免（全球互動最低） |
| 週日 10-11am | 可（假日晚起滑手機） |

注意：根據你的時區和受眾調整。以上數據基於全球 250 萬篇貼文分析。

## 誠實邊界

- 分數是基於公開演算法因子的「預測」，不是保證
- Threads 演算法持續變動，權重可能隨時調整
- 地區文化影響表現——全球策略不完全適用本地
- 研究中引用的部分互動倍數是近似值
- 此工具不能替代實際發文後觀察數據的回饋循環
- 不確定的時候，發就對了。真實數據永遠勝過預測
